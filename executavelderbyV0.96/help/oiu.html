Oracle Information Unit (OIU)

An OIU is a "self-contained" group of requirements.
An OIU is composed by a main rule and, possibly, a set of constraint rules and a set of safeguard rules. When analyzing the results, the oracle will produce the report based on different levels of veredict:
  • Success: if the main rule is true and the constraint rules, if any, are true
  • Constraint Failure: if the main rule is true and a constraint rule is false
  • Constraint Critical Failure: if the main rule is true, a constraint rule is false and a safeguard rule is false
  • Failure: if the main rule is false and the safeguard rules are true
  • Critical failure: if the main rule and a safeguard rule are false
An example of OIU may represent the following requirements:
  1. "If the input temperature is greater than 10°C, the controlled temperature must maintain at 10°C (main rule)"
  2. "The input temperature must be at maximum of 20°C (constraint rule)"
  3. "In case of controller failure, the controlled temperature must never be greater than 11°C (safeguard rule)"

Creating an OIU

To create an OIU:
  1. Go to the Module Editor tab (Figure 4)
  2. Select a pre-existing module by clicking on it at the Modules of Module... area or navigating the module tree at the left side of the screen. If no module was created, just a root module will exist
  3. Right-click at the Oracle Information Units of module... area to create an OIU

Defining an OIU

To define an OIU:
  1. Go to the Module Editor tab (Figure 4)
  2. Find the pre-created OIU on its respective module or create a new OIU as explained at the previous section
  3. Define the main rule:
    1. Press the Edit Main Rule button
      2. , at the upper-right in the OIU Editor screen
    2. Using the Rule Wizard: the wizard enables the user to construct rules as the following pattern: if "what is in the left side is true" it implies that "what is in the right side must be true"
      1. Defining the left side triad (property, behavior, temporal characteristic) of the rule:
        1. Select a property at its respective box. All the defined properties will be listed when clicking at it
        2. Select a behavior to the respective property, for example: "equals to" or "greater than"
          1. If there is no suitable behavior, you can create a new behavior. See how to create a new behavior
        3. Select a temporal characteristic, if needed OR the option <NONE>
        4. If the left side depends on more than one triad, you can add a new one by clicking on the + button
      2. Define the right side of the rule in the same way as it was done in the right side
      3. Press the Generate Expression button and the expression will be presented at the edition area in the middle of the screen
      4. Press the Save button, at the middle-right of the screen
    3. Defining the main rule manually. The user can define the main rule manually:
      1. PressEdit Main Rule
      2. Write the expression at the edition area in the middle of the screen
      3. More about the base specification language
      4. Press the Save button, at the middle-right of the screen
  4. Define the constraint rules:
    1. Press the Add Constraint button
    2. Write the constraint rule at the language editor box. More about the base specification language
    3. Write the respective meaning in natural language at the green box
    4. Press the Save button
  5. Define the saveguard rules:
    1. Press the Add Safeguard button
    2. Write the safeguard rule at the language editor box. More about the base specification language
    3. Write the respective meaning in natural language at the green box
    4. Press the Save button
