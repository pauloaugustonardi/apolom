The Behaviors

A behavior is a predictable relation of signal values
A behavior can, also, be used to express a function between values of different signals
Examples of behaviors are: greater than, ascendant and equals to
There are two main advantages on the creation of behaviors:
  • The user can reuse the behavior without the need to write it again. To reuse it, the user just need to select a behavior and relate it to a signal
  • The oracle generator pre-process the behavior before the OIU analysis. If a behavior A is related to a signal S and it is used on different OIUs, the analysis will be made just once, saving time

Creating a Behavior

Besides the built-in behaviors, the user can create new ones.
To create a new behavior:
  1. Open a project
  2. Go to the Behavior Editor tab (Figure 12)
  3. Select a base specification language at the behavior menu. For now, only TRIO is implemented
  4. Select the writing mode: Expression or Java code at the behavior editor area (for now, only Expression is implemented)
  5. Write the behavior at the edition area
  6. Select Save at the behavior menu
OBS: A helper for all the language features is accessible at the Global Objects area. Passing the mouse over each object, a detailed help is presented.

Deleting a Behavior

To delete a behavior:
  1. Go to the Behavior Editor tab (a project must be opened)
  2. Right-click over a behavior at the Behavior List and select delete behavior
