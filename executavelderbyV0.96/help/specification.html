The Specification

The specification is composed by Oracle Information Units (OIUs).
Each OIU is composed by a main rule, constraints rules and safeguard rules.
Each rule represents a relation between properties and its behaviors.
A property represents a line (signal) of the Simulink-like model.

To create the specification, the user should:
  • Create a new oracle project or open a previous one
  • Define the properties
  • Map the property to its respective line on the Simulink-like model
  • Modularize the specification, if desirable
  • Create an OIU within a module
  • Define the OIU
    • Defining a new behavior
  • Return to the main page
