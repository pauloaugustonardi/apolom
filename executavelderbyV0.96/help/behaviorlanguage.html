The Behavior Language

A behavior is written as an expression that returns true or false
At every simulation instant, the oracle analyzes if the behaviors are true or not
A behavior is written in the following pattern:
  • {some description in natural language} behavior_name(seq1,seq2,...;parameter1,parameter2,...) = <expression>
    • seq: is a generic name for a sequence of values of a signal from the start to the end of a simulation
    • parameter: the user can give a name that best describes a parameter
    • <expression>: an expression that returns true or false.

The Behavior Expression

The expression of a behavior can contain mathematical operators, relational operators, logical operators and navigators
The allowed mathematical operators are:
  • +
  • -
  • *
  • /
  • =
The allowed relational operators are:
  • >
  • >=
  • <
  • <=
  • ==
The allowed logical operators are:
  • AND
  • OR
  • NOT
  • ->
The navigators are:
  • Current(seq): returns the value of a sequence at the current analysis instant
  • Next(seq): returns the value of a sequence at the next analysis instant (w.r.t the current instant)
  • Next(seq,i): returns the value of a sequence at the analysis instant i in the future (w.r.t the current instant)
  • Previous(seq): returns the value of a sequence at the previous analysis instant (w.r.t the current instant)
  • Previous(seq,i): returns the value of a sequence at the analysis instant i in the past (w.r.t the current instant)
  • AtStart(seq): returns true if the analysis instant is the first instant
  • AtEnd(seq): returns true if the analysis instant is the last instant
  • Instant(seq): returns the current instant
  • Clock(seq): returns the analysis time at the current instant. Each instant may be attached to a simulation time. For example: instant 1 may represent 0.2s of simulation time
  • Instants(seq): returns the number of simulation instants

Behavior Examples

To a better visualization, the descriptions are in green, the behavior signature (name, sequences and parameters) is in blue and the expression is in red.
  • {The behavior is true when the current value from the sequence is greater than the parameter value} greaterthan(seq;value) = [Current(seq)>value]
  • {The behavior is true when the sequence mantains ascendant} ascendant(seq) = [NOT AtEnd(seq) AND Current(seq)<Next(seq)]
  • {The behavior is a function: true if seq1 is a multiple of seq2 defined by the parameter valor} function(seq1,seq2;valor) = [Current(seq1)==Current(seq2) * valor]
