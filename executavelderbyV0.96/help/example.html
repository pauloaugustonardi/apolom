A Complete Example

Creating a new project

  • Select New Project, as it is shown in Figure 1

    • Figure 1 - Creating a New Project
  • A new windows will be presente, as it is shown in Figure 2

    • Figure 2 - New Project Screen
  • Select a name, a path, a model and press create
    • The name must be unique. You can check if the name is unique by clicking on the check availability button
  • Back to the previous page
  • Return to the main page

Creating the specification attributes

  • Select the Instrumentation tab (Figure 3, A)
  • Type a property name at the name box (Figure 3, B)
  • Press the "add" button (Figure 3, C)

    • Figure 3 - Instrumentation Tab
  • Back to the previous page
  • Return to the main page

Mapping the specification to the model

  • In the Instrumentation tab, select the respective line (signal) from the model. It can be done by clicking on the respective line from the signal list or directly from the model (Figure 3, D)
  • Press the respective specification attribute at the Relation list (Figure 3, E)
  • Press the Insert button (Figure , F)
  • Back to the previous page
  • Return to the main page

Creating OIUs

  • Select the Module Editor tab (Figure 4, A)
  • Right-click within the "OIUs Area" (Figure 4, B) and select Insert New OIU

    • Figure 4 - Module Editor tab
  • Give a name (or a description) to the OIU and press ok (Figure 5)

    • Figure 5 - Selecting an OIU name
  • A new OIU is created (Figure 6)

    • Figure 6 - The created OIU
  • Back to the previous page
  • Return to the main page

Defining an OIU

  • In the Module Editor tab, select the OIU to be defined (Figure 6)
  • In the OIU Editor tab, press Edit Main Rule (Figure 7, A)
  • Create a rule using the Rule Wizard or by writing it on the specification code area (Figure 7, D). To use the Wizard:
    • Select a specification attribute, by pressing the Select a signal button (Figure 7, B) from the left side of the wizard expression and, then, by selecting an existent attribute from the list (Figure 7, C)

      • Figure 7 - OIU Editor tab
    • Select a behavior to the attribute, by pressing the Select a behavior button (Figure 8, A) from the left side of the wizard expression and, then, by selecting an existent behavior from the list (Figure 8, B)

      • Figure 8 - Selecting a behavior
    • If the behavior have a parameter, the respective value will be asked automatically, as it is shown in Figure 9, A
      Figure 9 - Setting a parameter
    • Select a shortcut, by pressing the Select a shortcut button from the left side of the wizard expression and, then, by selecting an existant shortcut (Figure 10, A)

      • Figure 10 - Selecting a shortcut
    • Use the same idea to complete the right side of the expression and press the Generate expression button (Figure 11, A). The expression will be generated as it is shown in Figure 11, B
    • Press the Save button (Figure 11, C)

      • Figure 11 - Generating a rule
  • To create constraints and safeguards, use the same steps as the main rule, by clicking in the Add Constraint and Add Safeguard buttons
  • Back to the previous page
  • Return to the main page

Defining behaviors

  • Select the Behavior Editor tab (Figure 12, A)
  • Select Expression or Java Code mode (Figure 12, B) - ONLY EXPRESSION IS IMPLEMENTED FOR NOW
  • Select the specification language (Figure 12, C) - ONLY TRIO IS IMPLEMENTED FOR NOW
  • Write the behavior (Figure 12, G), using the example in green as model, at the editor area (Figure 12, F)
    • All the navigators can be used (Figure 12, E)
  • Press Save on the menu behavior menu bar (Figure 12, D)
    • If there is any syntax error, it will be shown in the message area (Figure 12, H)
  • the new behavior will be listed at the Behavior List area (Figure 12, I)

    • Figure 12 - Behavior Editor tab
  • There is a Hover Help to each object (navigators, trio operators, behaviors) as it is shown in Figure 13

    • Figure 13 - Hover Help
  • Back to the previous page
  • Return to the main page

Executing the Oracle

  • Before executing the oracle, the user must execute the instrumented model on Simulink or another simulation tool
    • After the mapping between the specification properties and the model, a new instrumented model will be created in the project folder
  • After the model execution, select Execute on the option Execute in the menu bar (Figure 16, A)

    • Figure 16 - Execution options
  • Back to the previous page
  • Return to the main page

Reading the Report

  • The main list (Figure 17, A) presents all OIU that failed (C), which instant (D) and which rule was disrespected (E)
  • If a spurious value is detected, it is also presented (J)
  • Select a line from the main list to see a detailed report from the failure and the sequences of values of the related signals (B)
  • The detailed report presents the verdict level, the disrespected rule and the related signals
  • The instant is marked on the sequence of values of the related signals (i)

    • Figure 17 - The Oracle Report
